# Интернет-магазин «Чак-Чак»

Онлайн-магазин по продаже национальных татарских сладостей: чак-чак, баурсак, кош теле, татарский чай и другие традиционные угощения.

## О проекте

Проект представляет собой веб-приложение для продажи национальных татарских сладостей. Включает витрину товаров, корзину покупок, систему оформления заказов, личный кабинет пользователя и административную панель для управления товарами и заказами.

Целевая аудитория: жители Татарстана и других регионов России, туристы, а также организации для заказа сладостей на мероприятия.

## Архитектура

Проект построен на монолитной архитектуре с четким разделением на слои. Это обеспечивает упрощение разработки и развертывания, легкость тестирования и отладки, возможность быстрого внесения изменений и перспективу декомпозиции на микросервисы в будущем.

Основные части проекта:

- Фронтенд: пользовательский интерфейс, клиентская логика, адаптивный дизайн.
- Бэкенд: API слой, бизнес-логика, слой доступа к данным.
- База данных: хранение товаров, пользователей, заказов.
- Административная панель: управление каталогом, заказами, статистика.
- Система аутентификации: регистрация, вход, разграничение прав доступа.

## Технологии

Бэкенд реализован на Python 3.9+ с использованием фреймворка Django и Django REST Framework для создания REST API. В качестве базы данных используется PostgreSQL. Для кэширования и хранения сессий применяется Redis. Асинхронная обработка задач выполняется через Celery. Аутентификация реализована на JWT.

Фронтенд построен на HTML5, CSS3 и JavaScript (ES6+). Для адаптивного дизайна используется Bootstrap 5, для интерактивности — Vue.js. HTTP-запросы выполняются через Axios.

Для DevOps применяются Docker и Docker Compose для контейнеризации приложения, Nginx в качестве веб-сервера, GitHub Actions для CI/CD автоматизации. Хостинг планируется на AWS или Selectel.

Дополнительно используются Swagger/OpenAPI для документирования API, Pytest для тестирования, Black для форматирования кода, Pre-commit hooks для автоматической проверки, Sentry для отслеживания ошибок и Stripe/ЮKassa для платежной интеграции.

## База данных

Основные сущности базы данных:

- Пользователи: username, email, phone, address.
- Категории товаров: name, slug, description.
- Товары: связаны с категорией, содержат name, price, weight, composition, stock_quantity.
- Заказы: связаны с пользователем, содержат total_amount, status, delivery_address.
- Элементы заказа: связь заказа с товарами, quantity, price_at_time.
- Корзина: для неавторизованных пользователей с session_id и выбранными товарами.

## API Endpoints

Публичные эндпоинты:

- GET /api/products — список товаров с фильтрацией.
- GET /api/products/{slug} — детальная информация о товаре.
- GET /api/categories — список категорий.
- POST /api/cart — добавление в корзину.
- GET /api/cart — просмотр корзины.

Эндпоинты, требующие аутентификации:

- POST /api/orders — оформление заказа.
- GET /api/orders — история заказов.
- PUT /api/profile — обновление профиля.

Административные эндпоинты (требуют прав admin):

- POST /api/admin/products — добавление товара.
- PUT /api/admin/products/{id} — редактирование товара.
- DELETE /api/admin/products/{id} — удаление товара.
- GET /api/admin/orders — просмотр всех заказов.
- PUT /api/admin/orders/{id}/status — изменение статуса заказа.

## Интерфейс пользователя

Главная страница содержит баннер акций, популярные товары и категории.

Каталог товаров включает фильтрацию по цене и категории, сортировку, поиск и карточки товаров.

Страница товара содержит детальное описание, фотографии, состав и кнопку добавления в корзину.

Корзина отображает список товаров с возможностью изменения количества, ввода промокодов и показывает итоговую стоимость.

Оформление заказа включает форму с контактными данными, выбор способа доставки и оплаты.

Личный кабинет содержит историю заказов, статусы, личные данные и избранные товары.

## Безопасность

Безопасность обеспечивается через HTTPS для шифрования данных, хеширование паролей (bcrypt/Argon2), CSRF-защиту, предотвращение SQL-инъекций через использование ORM и параметризованных запросов, XSS-защиту экранированием пользовательского ввода, rate limiting для ограничения запросов, валидацию данных на сервере и клиенте, а также регулярное резервное копирование базы данных.

## CI/CD Pipeline

Процесс разработки организован на GitHub с использованием веток main (стабильная версия), develop (разработка и тестирование), feature/* (новые функции) и hotfix/* (срочные исправления).

Разработчик создает ветку от develop, вносит изменения, создает Pull Request в develop, проходит код-ревью и автоматическое тестирование через GitHub Actions, после чего изменения мержатся. Периодически develop мержится в main для создания релиза.

Автоматизация включает сборку проекта, запуск unit-тестов, проверку стиля кода и безопасности зависимостей, а также автоматический деплой на тестовый сервер при пуше в develop и на production при пуше в main.

Тестирование охватывает unit-тесты, интеграционные и end-to-end тесты.

## Запуск проекта

Клонирование репозитория:

git clone https://github.com/AdelinaKiseleva/chak-chak-shop.git
cd chak-chak-shop

Запуск микросервисов:

docker-compose up -d

Запуск основного приложения:

cd backend
python manage.py runserver
